# tailor-cv — Installation

This skill uses a fixed folder layout. Projects must conform to the structure below. Paths are not configurable.

## Required structure

```
<project-root>/
  source/
    identity/
      SOUL.md          ← voice, tone, values, content boundaries
      IDENTITY.md      ← candidate name, role, handle, location, language
      AGENTS.md        ← CV tracks table + session contract (skill entry point)
    memory/
      MEMORY.md        ← memory index (one line per memory file)
      *.md             ← individual memory files referenced by MEMORY.md
    soft-skills/
      *.md             ← cross-track soft skills (optional; folder may be absent)
    tracks/
      <track>/
        <cv-file>.md   ← CV file for this track
        output.md      ← generated by skill on each run (overwritten)
        <Co>_<Role>.md ← job offer saved verbatim on each run
  .agents/
    skills/
      tailor-cv/
        SKILL.md       ← this skill
        INSTALL.md     ← this file
```

## File contracts

### source/identity/AGENTS.md — required

Must contain a `## CV Tracks` section with a markdown table. The skill reads this table to present track options and resolve file paths. Minimum columns: Track, File, Use when.

```markdown
## CV Tracks

| Track | File | Use when |
|---|---|---|
| Track A | source/track-a/candidate_trackA_CV.md | When to use track A |
| Track B | source/track-b/candidate_trackB_CV.md | When to use track B |
```

### source/identity/SOUL.md — required

Defines authentic voice, tone, and content boundaries. The skill enforces SOUL.md style on all output — it will not adopt the job description's language. Must include at minimum: tone rules, what the voice sounds like (yes/no examples), and content boundaries (what to never claim).

### source/identity/IDENTITY.md — required

Candidate metadata used for proofreading and localization. Must include at minimum: name, role, language (e.g. `en-US`).

### source/memory/MEMORY.md — required

Index of memory files. One entry per file. The skill reads this at session start and loads all listed files. Format:

```markdown
- [Label](filename.md) — one-line description
```

### source/soft-skills/ — optional

If present, the skill reads it when the JD signals collaboration, leadership, or stakeholder framing. If absent, the skill skips it silently.

## Quickstart

A skeleton of the required structure is in `template/`. Copy it to your project root and fill in the placeholders:

```
cp -r .agents/skills/tailor-cv/template/. <your-project-root>/
```

Then follow the file contracts below before running the skill.

## Registering the skill in Claude Code

Add the skill to your project's `.claude/settings.json`:

```json
{
  "skills": [".agents/skills/tailor-cv/SKILL.md"]
}
```

Or install globally in `~/.claude/settings.json` to use across projects.

Invoke with:

```
/tailor-cv
```

## Verification checklist

Before first run, confirm:

- [ ] `source/identity/AGENTS.md` exists and has a `## CV Tracks` table
- [ ] `source/identity/SOUL.md` exists with tone and boundary rules
- [ ] `source/identity/IDENTITY.md` exists with name and language
- [ ] `source/memory/MEMORY.md` exists (may be empty index)
- [ ] All CV files listed in the tracks table exist at their stated paths
